Historia de los Sistemas Operativos

Desde los primeros sistemas de lotes hasta la era moderna de la computación en la nube

1950s

Sistemas de Lotes (Batch)

Los primeros sistemas operativos procesaban trabajos en grupos (lotes). Los usuarios entregaban tarjetas perforadas y esperaban horas o días por los resultados.

  • Procesamiento secuencial
  • Sin interacción usuario-sistema
  • Uso intensivo de tarjetas perforadas
1969

Unix

Desarrollado en Bell Labs por Ken Thompson y Dennis Ritchie. Introdujo conceptos revolucionarios como el sistema de archivos jerárquico y la filosofía "todo es un archivo".

  • Multiusuario y multitarea
  • Shell y pipes
  • Portabilidad del código
1981

MS-DOS

Microsoft Disk Operating System se convirtió en el estándar para PCs IBM. Interfaz de línea de comandos que sentó las bases del scripting batch moderno.

  • Interfaz CLI pura
  • Sistema de archivos FAT
  • Primeros archivos .bat
1984-1991

Mac OS y Windows

La revolución de las interfaces gráficas (GUI). Apple lanzó el Macintosh (1984) y Microsoft respondió con Windows 3.0 (1990) y el revolucionario Windows 95.

  • Interfaz gráfica intuitiva
  • Multitarea cooperativa/preventiva
  • Accesibilidad para masas
1991

Linux

Linus Torvalds crea el kernel Linux. Sistema operativo libre y de código abierto que revolucionó servidores, embebidos y supercomputación.

  • Código abierto
  • Altamente personalizable
  • Dominio en servidores
2000s+

Cloud y Móvil

iOS (2007), Android (2008) y el auge de la computación en la nube. Los sistemas operativos se vuelven distribuidos, virtualizados y orientados a servicios.

  • Virtualización y contenedores
  • Computación distribuida
  • SO como servicio (SaaS)

Menú del script de mantenimiento

El archivo MantenimientoV2.bat incluye un menú interactivo donde el usuario puede elegir qué tarea ejecutar. Utilizamos set /p para leer la opción y call para ejecutar otros scripts como Scanner.bat.

@echo off :menu echo 1 Limpieza echo 2 Scanner set /p op= if "%op%"=="2" call Scanner.bat

Uso de Netstat

El comando netstat -ano muestra todas las conexiones activas del equipo. Es útil para detectar programas sospechosos o problemas de red. Guardamos el resultado en un archivo:

netstat -ano > conexiones.txt

Fundamentos Básicos de Batch

En este nivel aprenderás los conceptos esenciales para crear tus primeros scripts.

1. Estructura básica
@echo off title "Mi Primer Script" echo "Hola Mundo" pause
2. Variables Simples

3. Comandos Esenciales
  • echo - Muestra mensajes
  • cls - Limpia la pantalla
  • pause - Espera tecla
  • dir - Lista archivos
  • cd - Cambia directorio

Batch - Nivel Intermedio

Scripts Avanzados y Estructuras de Control

1. Condicionales IF-ELSE
if %edad% geq 18 ( echo "Eres mayor de edad" ) else ( echo "Eres menor de edad" ) if exist "archivo.txt" del "archivo.txt"
2. Bucles FOR
:: Bucle numérico for /L %%i in (1,1,10) do ( echo "Iteración %%i" ) :: Bucle en archivos for %%f in (*.txt) do ( echo "Procesando: %%f" )
3. Funciones (CALL)
call :funcion_saludo "Juan" goto :eof :funcion_saludo echo "Hola, %~1" exit /b

Script de Mantenimiento Avanzado

Análisis completo línea por línea de un script profesional de mantenimiento de Windows

Script Completo: Mantenimiento_Windows.bat

Línea 1
@echo off

@ - Oculta la visualización del comando mismo.
echo off - Desactiva la visualización de todos los comandos subsiguientes, manteniendo la salida limpia para el usuario.

Línea 2
setlocal EnableDelayedExpansion

setlocal - Inicia un entorno local para variables (se restauran al finalizar).
EnableDelayedExpansion - Permite usar variables dentro de bucles con sintaxis !variable! en lugar de %variable%, evaluándose en tiempo de ejecución, no de parseo.

Línea 3-4
title "Mantenimiento del Sistema v2.0" color 0A

title - Establece el título de la ventana de la consola.
color 0A - Cambia los colores: 0 = fondo negro, A = texto verde brillante (estilo Matrix/profesional).

Línea 5-8
set "logFile=%temp%\Mantenimiento_%date:~-4,4%%date:~-10,2%%date:~-7,2%.log" echo "========================================" >> "%logFile%" echo "Inicio de mantenimiento: %date% %time%" >> "%logFile%" echo "========================================" >> "%logFile%"

set "logFile=..." - Define ruta del archivo log en %temp% con fecha en nombre (YYYYMMDD).
%date:~-4,4% - Manipulación de substring: extrae 4 caracteres empezando desde el final-4 (año).
>> - Redirección de salida: append (añade al final sin sobrescribir).

Línea 9-15
echo "[1/5] Limpiando archivos temporales..." del /q /f /s %temp%\*.tmp 2>nul del /q /f /s %windir%\temp\*.* 2>nul echo "Limpieza completada" >> "%logFile%"

del /q /f /s - Elimina archivos: /q (quiet, sin confirmar), /f (force, archivos protegidos), /s (subdirectorios).
2>nul - Redirige errores (stderr, stream 2) a nul (dispositulo nulo), ocultando mensajes de error.
%windir% - Variable de entorno que apunta a C:\Windows.

Línea 16-24
echo "[2/5] Verificando integridad del disco..." echo "y" | chkdsk C: /f /scan if %errorlevel% equ 0 ( echo "Disco OK" >> "%logFile%" ) else ( echo "ADVERTENCIA: Errores encontrados" >> "%logFile%" )

echo "y" | - Pipe: envía "y" como respuesta automática al comando chkdsk (confirma operación).
chkdsk C: /f /scan - Verifica disco C: /f (corrige errores), /scan (escaneo online sin desmontar).
%errorlevel% - Variable automática con código de retorno del último comando (0 = éxito).

Línea 25-35
echo "[3/5] Escaneando archivos del sistema..." sfc /scannow > "%temp%\sfc_output.txt" findstr /c:"no encontró" "%temp%\sfc_output.txt" > nul if %errorlevel% equ 0 ( echo "SFC: No se encontraron violaciones" >> "%logFile%" ) else ( echo "[4/5] Reparando imagen del sistema con DISM..." DISM /Online /Cleanup-Image /RestoreHealth )

sfc /scannow - System File Checker: escanea y repara archivos protegidos del sistema.
findstr /c:"texto" - Busca cadena literal en archivo. Si encuentra, errorlevel=0.
DISM - Deployment Image Servicing and Management: repara la imagen de Windows usando Windows Update.

Línea 36-45
echo "[5/5] Optimizando configuracion de red..." ipconfig /flushdns ipconfig /release ipconfig /renew echo "Red optimizada" >> "%logFile%" echo "========================================" >> "%logFile%" echo "Mantenimiento completado: %date% %time%" >> "%logFile%" echo "Log guardado en: %logFile%" pause

ipconfig /flushdns - Vacía caché DNS resolviendo problemas de resolución de nombres.
/release y /renew - Libera y renueva la configuración IP DHCP.
pause - Espera pulsación de tecla antes de cerrar, permitiendo ver resultados.

Conceptos Clave Aprendidos

  • Redirección: > (sobrescribir), >> (añadir), 2>nul (ocultar errores)
  • Pipes: | (pasar salida a entrada de otro comando)
  • Variables de entorno: %temp%, %windir%, %date%, %time%
  • Control de flujo: if/else, errorlevel, goto
  • Logging: Técnica esencial para scripts de producción
